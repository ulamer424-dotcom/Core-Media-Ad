# CoreMedia Ad Subscription Manager

نظام عربي متجاوب لإدارة الاشتراكات الشهرية والسنوية باستخدام HTML وCSS وJavaScript وLocal Storage فقط.


## تحديث V2 — صفحة تسجيل الدخول الإلزامية

- تظهر صفحة تسجيل الدخول دائمًا عند فتح الموقع أو تحديثه.
- لا توجد خاصية تجاوز تلقائي بسبب جلسة محفوظة أو خيار «تذكّرني».
- بعد إدخال البيانات الصحيحة ينتقل المستخدم مباشرة إلى لوحة التحكم.
- بيانات الدخول الافتراضية:

```text
اسم المستخدم: admin
كلمة المرور: CoreMedia@2026
```

يمكن تغيير اسم المستخدم وكلمة المرور من **الإعدادات ← بيانات تسجيل الدخول**.

## التشغيل

1. فك ضغط الملف.
2. افتح مجلد `subscription-manager`.
3. افتح `index.html` في المتصفح.
4. للحصول على تجربة أفضل مع إشعارات المتصفح، شغّل الموقع عبر HTTPS بعد رفعه، أو عبر خادم محلي مثل Live Server في VS Code.

لا يحتاج المشروع إلى npm أو React أو قاعدة بيانات.


## تسجيل الدخول

عند فتح الموقع تظهر صفحة تسجيل الدخول أولًا، وبعد إدخال البيانات الصحيحة ينتقل المستخدم إلى لوحة التحكم.

بيانات الدخول الافتراضية:

```text
اسم المستخدم: admin
كلمة المرور: CoreMedia@2026
```

يمكن تغيير اسم المستخدم وكلمة المرور من **الإعدادات ← بيانات تسجيل الدخول**، ثم الضغط على **تسجيل الخروج** لاختبار البيانات الجديدة. خيار **تذكّر تسجيل الدخول** يبقي الجلسة محفوظة على الجهاز، بينما يؤدي زر تسجيل الخروج إلى حذفها.

لأن المشروع موقع ثابت، فإن نظام الدخول حماية محلية فقط ويمكن الاطلاع على منطق التحقق من خلال ملفات الموقع. للأمان الحقيقي يلزم Backend وقاعدة بيانات وجلسات آمنة.

## أهم الوظائف

- إضافة وتعديل وحذف واستعادة الاشتراكات.
- تجديد الاشتراك مع سجل كامل لعمليات التجديد.
- حساب الوقت المتبقي والحالة تلقائيًا.
- تنبيهات عند 10 و5 و3 و1 و0 أيام.
- تنبيه دوري كل 10 أيام عند فتح الموقع.
- زر واتساب برسالة مشفرة وجاهزة.
- بحث وفلاتر وترتيب.
- تقويم شهري.
- رسوم بيانية عبر Chart.js.
- تصدير CSV وJSON وطباعة.
- نسخ احتياطي واستيراد.
- سلة محذوفات.
- الوضع الداكن.
- قفل اختياري برمز PIN وقفل تلقائي.
- إخفاء البيانات الحساسة عند الطباعة.

## تغيير رقم واتساب

1. افتح الموقع.
2. انتقل إلى **الإعدادات**.
3. غيّر الرقم في حقل **رقم واتساب الدولي داخل رابط wa.me**.
4. اكتب الرقم بدون علامة `+` وبدون الصفر الأول.

الرقم الحالي:

```text
201125643485
```

ويمكن تغييره برمجيًا داخل `script.js` في:

```javascript
whatsappNumber: "201125643485"
```

## إضافة اشتراك جديد

اضغط **إضافة اشتراك** من الهيدر أو من الإجراءات السريعة، ثم أكمل الحقول المطلوبة واضغط **حفظ الاشتراك**. تُحفظ البيانات في Local Storage داخل المتصفح نفسه.

## رفع المشروع على GitHub Pages

1. أنشئ مستودعًا جديدًا على GitHub.
2. ارفع محتويات مجلد `subscription-manager` إلى جذر المستودع، بحيث يكون `index.html` في الجذر.
3. احفظ التغييرات عبر Commit.
4. افتح **Settings → Pages**.
5. تحت **Build and deployment** اختر **Deploy from a branch**.
6. اختر فرع `main` والمجلد `/root` ثم احفظ.

## ربطه بـ Cloudflare Pages

1. ادخل إلى لوحة Cloudflare.
2. افتح **Workers & Pages** وأنشئ مشروع Pages.
3. اختر الربط مع GitHub وحدد المستودع.
4. لأن المشروع HTML/CSS/JS ثابت:
   - Framework preset: `None`
   - Build command: اتركه فارغًا
   - Build output directory: `/`
5. ابدأ النشر. أي Push جديد إلى GitHub سيؤدي إلى نشر نسخة جديدة تلقائيًا.

## قيود واتساب والتنبيهات

هذا مشروع ثابت يعمل داخل المتصفح؛ لذلك لا يستطيع إرسال رسالة واتساب تلقائيًا عندما يكون الموقع مغلقًا. النسخة الحالية تنفذ:

- تنبيه داخل الموقع.
- Browser Notification بعد موافقة المستخدم.
- زر يفتح رسالة واتساب جاهزة.
- حفظ تاريخ آخر تنبيه في Local Storage.

للإرسال التلقائي الحقيقي يلزم Backend آمن، مثل Node.js أو Firebase Functions أو Cloudflare Workers، مع Cron Job وربط WhatsApp Business Cloud API. لا تضع مفاتيح WhatsApp أو Access Tokens داخل `script.js`.

## الأمان

Local Storage ليس مخزنًا آمنًا لكلمات المرور الحقيقية. رمز PIN الموجود في المشروع حماية محلية بسيطة وليس بديلًا عن نظام تسجيل دخول. النسخة الإنتاجية تحتاج Backend وقاعدة بيانات وتشفير وتسجيل دخول وصلاحيات وHTTPS.

## ملاحظات

- البيانات التجريبية تُنشأ مرة واحدة، ويمكن حذفها من زر **حذف البيانات التجريبية**.
- حذف الاشتراك ينقله إلى سلة المحذوفات أولًا.
- الاستيراد يستبدل البيانات الحالية بعد رسالة تأكيد.
- يفضّل الاحتفاظ بنسخة احتياطية منتظمة.
